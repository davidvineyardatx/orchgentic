# Roadmap Note — Token-Aware Orchestration

v0.8.0-beta.3 established Token Intelligence and Local Reasoning Proof.

Key strategic direction:

```text
Use deterministic routing first.
Use local reasoning second.
Use local LLMs for low-risk planning and classification.
Use external LLMs only when quality, complexity, or uncertainty requires escalation.
```

Near-term priorities after beta.3:

- runtime hardening and error consistency
- configuration contract freeze
- tool/plugin contract stabilization
- local LLM provider path for planner, reviewer, and classification tasks
- richer local LLM eligibility controls
- workflow layer foundation
- RAG, memory, and knowledge stabilization
