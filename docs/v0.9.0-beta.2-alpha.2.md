# v0.9.0-beta.2-alpha.2

## Focus

Runtime-facing doctor integration for the existing RAG, memory, and provider stabilization checks.

This alpha does not add new RAG, memory, provider, or workflow features. It wires the beta.2 checks closer to the developer experience so configuration failures are easier to diagnose before agent execution.

## Added

- `Beta2DoctorResult`
- `doctor_agent_layers`
- `format_beta2_doctor_text`
- YAML path support for agent layer doctor checks
- JSON output support through `Beta2DoctorResult.to_json()`
- Optional Typer command module: `agent_core/cli/doctor_beta2.py`
- Regression tests for doctor result shape and readable failure output

## Example usage from Python

```python
from agent_core.stabilization import doctor_agent_layers, format_beta2_doctor_text

result = doctor_agent_layers("agents/bob.yaml")
print(format_beta2_doctor_text(result))
```

## Example CLI wiring

The included command module can be attached to the main CLI if the project already has a root doctor command.

```python
from agent_core.cli.doctor_beta2 import app as beta2_doctor_app

app.add_typer(beta2_doctor_app, name="doctor")
```

Then:

```bash
orch doctor agent agents/bob.yaml
orch doctor agent agents/bob.yaml --json
```
