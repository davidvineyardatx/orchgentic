# v0.9.0-beta.2-alpha.3

## Focus

Runtime boundary hardening for the existing provider, memory, and knowledge/RAG layers.

This alpha keeps beta.2 focused on stabilization. It does not add new provider families, RAG architecture, workflow features, or dashboard functionality.

## Added

- `AgentLayerStabilizationError`
- `assert_agent_layers_ready`
- `build_runtime_failure_message`
- `agent_core/runtime/stabilization_gate.py`
- Runtime regression tests for fail-fast behavior

## Why this matters

Before this alpha, the stabilization checks could be run manually or through doctor-style helpers. This alpha adds a small runtime-facing boundary helper so an agent run can fail before attempting to initialize broken provider, memory, or knowledge config.

## Intended integration point

Call this before agent execution initializes providers, memory, or knowledge.

```python
from agent_core.runtime.stabilization_gate import validate_agent_runtime_layers

validate_agent_runtime_layers(agent.config, project_root=project_root)
```

## Failure example

```text
Agent layer stabilization failed for Bob.
Fix the following configuration errors before running the agent:
- [provider] provider.model.missing at provider.model: Provider model is required.
- [memory] memory.db_path.missing at memory.db_path: Memory db_path is required for local/SQLite memory.
```
