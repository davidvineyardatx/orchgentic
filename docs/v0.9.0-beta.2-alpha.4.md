# v0.9.0-beta.2-alpha.4

## Focus

Runtime regression coverage for the existing provider, memory, and knowledge/RAG layers.

This alpha is stabilization-only. It does not add new workflow features, new provider families, new RAG architecture, hosted services, or dashboard functionality.

## Added

- `Beta2RegressionResult`
- `normalize_search_result`
- `check_empty_search_result`
- `check_provider_factory_result`
- Runtime regression tests for:
  - empty memory search result normalization
  - empty knowledge search result normalization
  - provider factory object sanity
  - invalid config fail-fast behavior
  - disabled memory/knowledge behavior
  - SQLite path creation after validation

## Why this matters

The earlier beta.2 alphas made configuration failures clearer. This alpha adds regression coverage for the safe behavior expected around the existing runtime layers.

The tests intentionally avoid external LLM calls and external vector database connections.
