# v0.9.0-beta.2 Stabilization

## Focus

v0.9.0-beta.2 stabilizes the existing RAG, memory, and provider layers.

The goal is hardening, not expansion.

## Completed in this release

### alpha.1 — Pure stabilization checks

- Provider configuration validation.
- Memory configuration validation.
- Knowledge/RAG configuration validation.
- Combined agent layer check helper.
- Regression tests for common configuration failures.

### alpha.2 — Doctor integration

- Doctor-style result object.
- Human-readable doctor output.
- JSON output support.
- YAML path loading for agent configs.
- Optional Typer command module for `orch doctor agent <path>`.

### alpha.3 — Runtime boundary hardening

- Fail-fast runtime helper.
- Runtime-facing stabilization error.
- Stable wrapper at `agent_core.runtime.stabilization_gate`.
- Runtime tests for clear failure messages.

### alpha.4 — Runtime regression coverage

- Search result normalization helpers.
- Safe empty memory/RAG search checks.
- Provider factory sanity checks.
- SQLite local path creation tests.
- Disabled memory/knowledge non-blocking tests.

## Non-goals

This release intentionally avoids:

- New workflow features.
- New RAG architecture.
- New provider families.
- Hosted dashboard work.
- No-code builder work.
- Marketplace work.
- Complex distributed runtime work.
