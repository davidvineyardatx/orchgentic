# v1.0.1 Cleanup Build

## Goal

Give users a clean codebase after cloning the repo.

This cleanup removes local/generated state and release-package-only validation artifacts while preserving the actual product code and useful product test suite.

## What should be removed

### Release-only tests

These tests validated release packaging and documentation files during v1 preparation. They are not useful starter tests for cloned user repos.

```text
tests/test_rc1_docs_exist.py
tests/test_rc1_clean_install_commands_documented.py
tests/test_v1_release_docs.py
```

### Local runtime state

Local runtime state should not be committed or shipped as starter data.

```text
memory/*
knowledge/*
```

Keep placeholders:

```text
memory/.gitkeep
knowledge/.gitkeep
```

### Build/cache artifacts

```text
.pytest_cache/
.mypy_cache/
.ruff_cache/
__pycache__/
dist/
build/
*.egg-info/
```

### Release-package snippets/manifests

These can be removed from the repo root after release packaging:

```text
manifest.json
README_RC1_SNIPPET.md
README_V1_SNIPPET.md
README_BETA2_SNIPPET.md
```

Version-specific changelog/release-note files can either be kept in the repo or moved to GitHub releases. The cleanup script removes them by default, but supports `--keep-release-notes`.

## What should stay

Keep real framework tests, including:

- core runtime tests
- tool runtime tests
- provider tests
- memory tests
- knowledge/RAG tests
- reasoning tests
- workflow contract tests
- stabilization tests that protect runtime behavior

Do not remove the whole `tests/` directory.

## Recommended cleanup command

Preview first:

```bash
python scripts/cleanup_release_artifacts.py --dry-run
```

Apply:

```bash
python scripts/cleanup_release_artifacts.py
```

Keep release notes/changelogs:

```bash
python scripts/cleanup_release_artifacts.py --keep-release-notes
```

## Recommended validation after cleanup

```bash
python -m pytest -q
orch --help
orch init
orch create-agent Bob
orch list-agents
orch list-tools
```

## Expected result

A cloned repo should feel like a clean developer starter project:

- no generated memory database
- no generated knowledge database
- no release packaging leftovers
- no cache/build artifacts
- product tests still available
- docs still usable
